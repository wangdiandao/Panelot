# Security Policy

## Supported versions

Security updates are provided for the latest published Panelot release.

## Reporting a vulnerability

Please use GitHub private vulnerability reporting for this repository. Do not open a public issue for an unpatched vulnerability and do not include real credentials or private browsing data in a report.

Include the affected version, impact, reproduction steps, and any suggested mitigation. We aim to acknowledge a report within 5 business days and will coordinate disclosure after a fix is available.

Panelot can send user-selected content to configured model providers and MCP servers. Reports involving a third-party service should distinguish Panelot behavior from that service's own data handling.
