# AGENTS.md

面向在本仓库工作的 AI 协作者（Claude Code 等）的项目说明。

## 项目速览

Panelot 是一款 Chrome（MV3）浏览器 Agent 扩展（WXT + React + TypeScript）。设计规范在 [DESIGN.md](./DESIGN.md) 与 [docs/](./docs)；改行为前先读对应分章。

常用命令：

```bash
pnpm compile   # tsc --noEmit（类型检查）
pnpm test      # Vitest 单测
pnpm e2e       # Playwright e2e
pnpm build     # 生产构建
```

注意：改引擎（后台 Service Worker）代码后，刷新页面不生效，需到 `chrome://extensions` 重载扩展。

## 代码卫生规则（AI 协作必读）

### 开发过程中禁止

- 禁止写版本迭代注释（v1/v2/v3、「新版/旧版」、「优化后」）和修复轮次标注（P1/P2、C1/C2、FIX-N、「第二轮修复」）。变更历史交给 git 记录。
- 注释只解释「为什么这样写」（约束、坑、权衡），禁止写「本次改了什么」式的变更叙事。
- 禁止整段注释掉旧代码「以备回滚」，直接删除，需要时从 git 找回。
- 临时脚本和调试文件统一放入 scratch/ 目录（已在 .gitignore 中），不要散落在仓库其它位置。

### 每次任务完成后自查（提交前必做）

1. 删除本次任务创建的临时验证/调试脚本；有回归价值的先改写为正式测试用例再删除原件。
2. 删除遗留的调试语句（console.log / print / dbg 等）和被注释掉的代码块。
3. 检查本次改动涉及的注释，清除版本标注、修复轮次标注和变更叙事型注释。
4. 本次任务生成的总结、修复报告等临时文档，除非用户明确要求保留，不要提交进仓库。
5. 规划类文档（PLAN/TODO/ROADMAP）中本次已完成的条目，及时删除或标记。
6. 以上清理与功能改动放在同一次提交或紧随其后的独立提交中，不留到下次。

### 例外（必须保留）

API 路由版本（如 `/api/v1`）、语义化版本号、协议/数据格式版本号、仍然有效的 TODO、解释「为什么这样写」的注释、许可/版权注释。
