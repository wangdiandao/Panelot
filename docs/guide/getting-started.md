# 安装与首次配置

## 系统要求

- Chrome 116 或更高版本，或相应版本的 Microsoft Edge。
- 一个可从浏览器访问的 OpenAI 兼容接口或 Anthropic 接口。
- 对应服务要求的 API Key；明确允许匿名访问的端点可以留空（常见于本地模型服务）。

Panelot 目前通过 GitHub Release 提供开发者模式安装包，尚无可确认的浏览器商店安装渠道。

## 安装扩展

1. 前往 Panelot 的 [GitHub Releases](https://github.com/wangdiandao/Panelot/releases)，Chrome 下载 `panelot-chrome.zip`，Edge 下载 `panelot-edge.zip`。
2. 将 ZIP 解压到一个长期保留的目录。加载后不要移动或删除该目录。
3. Chrome 打开 `chrome://extensions`，Edge 打开 `edge://extensions`。
4. 开启“开发者模式”，选择“加载已解压的扩展程序”，再选择刚才的解压目录。
5. 将 Panelot 固定到浏览器工具栏。点击图标或按 `Alt+P` 打开侧边栏。

升级手动安装的版本前，先在“设置 → 数据”导出备份。保留原来的扩展目录路径，停用扩展或关闭相关浏览器窗口后，用新版本的完整文件替换旧文件，再到扩展管理页点击 Panelot 的“重新加载”。

“设置 → 关于 → 检查更新”会对比当前版本与最新的 GitHub Release。发现新版本时，可以直接下载当前浏览器对应的 ZIP；下载完成后仍需按上面的步骤覆盖原目录并重新加载扩展。

不要混用新旧版本文件，也不要直接改为加载另一个目录。未打包扩展的身份与目录有关；换目录后，浏览器可能把它识别为一份新安装，原有本地数据也不会出现在其中。

## 添加第一个模型连接

首次打开且尚未配置模型时，Panelot 会显示引导。也可以随时进入“设置 → 模型”完成同样的配置。

1. 选择接口类型：
   - **OpenAI 兼容**：端点需要提供兼容的 `/chat/completions` 流式接口。
   - **Anthropic**：端点需要提供兼容的 `/v1/messages` 流式接口。
2. 输入 Base URL 和 API Key。远程端点必须使用 HTTPS；HTTP 仅允许 `localhost`、`127.0.0.1` 或 `[::1]`。
3. 点击“验证连接”。浏览器可能先询问是否允许 Panelot 访问该接口域名，这是向你配置的服务发送请求所必需的站点访问权限。
4. 检查“可达、Key 有效、流式、工具调用”等结果。验证失败时不要仅保存后直接假定可用。
5. 保存连接，并在模型页选择一个“默认模型”。若端点不支持模型列表接口，可在连接中手动填写模型 ID，每行一个。

验证会向端点发送一组最小请求，检查网络、认证、流式响应和工具调用结构。验证通过只说明这些基础环节可用，不代表该端点的所有模型、参数和长对话都已兼容。保存后再发送一次真实对话进行确认。

更完整的连接选项见 [Provider 与模型](./providers-and-models.md)。

## 选择默认权限模式

引导的第二步会让你选择浏览器操作模式：

- **全程询问**：读取页面和执行操作前都询问。
- **操作询问**：允许读取；点击、输入、下载等操作默认先询问。这是默认选项。
- **自动操作**：普通操作可自动执行，但敏感站点、疑似敏感信息和权限规则仍会限制或要求确认。

第一次使用时先保留“操作询问”。熟悉审批信息后，再按任务调整。每个对话也能从输入框临时切换模式；详细规则见[浏览器操作与权限](./browser-and-permissions.md)。

## 完成一次测试

1. 打开一个普通 `https://` 网页。
2. 打开 Panelot 侧边栏，点击当前页面提示中的“附着到对话”，或在输入框键入 `@` 后选择页面。
3. 输入“总结这个页面的主要内容”，然后发送。
4. 如果浏览器询问站点访问，请核对域名后授权。

侧边栏展示当前页面并不等于已经把页面内容加入对话。只有出现页面上下文胶囊后，该次发送才会包含所选页面正文。
