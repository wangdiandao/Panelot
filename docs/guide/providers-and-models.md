# Provider 与模型

Panelot 直接连接你配置的模型服务。它不代理请求、不提供共享 Key，也不替模型服务承诺可用性、价格或隐私政策。

## 支持的接口类型

Panelot 支持两种连接类型：

- **OpenAI 兼容**：使用兼容的流式 Chat Completions 和工具调用格式。
- **Anthropic**：使用兼容的 Messages 流式接口和工具调用格式。

Gemini、DeepSeek、Ollama 等名称本身不是独立接口类型。如果服务提供 OpenAI 兼容层，可以按其文档填写 OpenAI 兼容连接；能否正常工作以“验证连接”和真实请求结果为准。

## 添加和验证连接

在“设置 → 模型”选择“添加连接”，填写：

- **接口类型**：必须与端点实际协议一致。
- **名称**：可选；留空时使用端点域名。
- **Base URL**：远程地址必须是 HTTPS；本地回环地址可用 HTTP。OpenAI 兼容端点通常带 `/v1`，但某些代理或云接口使用其它路径，应以服务文档为准。
- **API Keys**：每行一个。Panelot 会继续使用当前可用 Key，遇到认证或限流错误时再尝试后续 Key，并非逐请求轮询。
- **模型列表**：端点无法返回模型列表时，可手工填写模型 ID，每行一个。

点击“验证连接”后，Panelot 会检查端点可达性、Key、流式响应和工具调用结构，并尝试获取模型列表。验证前可能需要授予端点域名的站点访问权限。

保存并启用连接后，还需要在模型页选择一个默认模型。删除连接不会删除已有会话文字，但该连接的模型将不能再用于新请求。

## 兼容性选项

OpenAI 兼容端点的细节并不统一。只有在服务文档或错误信息明确要求时，才调整连接中的兼容性选项，例如：

- 不发送流式 usage 选项；
- 从 `<think>` 标签读取推理内容；
- 强制每次只调用一个工具；
- 不使用 `system` 角色；
- 改用 `max_completion_tokens` 字段。

修改后重新运行验证并发送一次真实请求。若端点返回不完整流、未知结束原因或无法解析的工具调用，Panelot 会按协议错误停止，不会把残缺响应当作完整成功。

高级设置还允许添加自定义 Header，以及用 JSON 描述模型能力和价格。这些字段面向了解上游接口的用户；错误的能力标注可能让模型收到它实际不支持的图片、工具或参数。自定义 Header 中的敏感值会在本机加密保存。

## 模型选择

全局默认模型用于选择“默认模型”的对话。也可以从顶部或输入框的模型选择器临时指定连接和模型；该选择用于接下来的任务，并作为以后打开对话时优先采用的最近模型。历史消息仍会记录当时实际使用的模型。

判断模型是否适合浏览器任务，先看四点：

- 能否稳定输出结构化工具调用；
- 是否支持任务所需的图片输入；
- 上下文窗口能否容纳会话和页面内容；
- 端点是否正确实现流式结束信号。

“验证连接”通过不代表某个具体模型在长任务中一定可靠。执行重要任务前，先在范围较小的页面上试跑。

## 模型预设

“设置 → 预设”可以把以下设置组合为可复用预设：

- 连接和模型；
- 系统提示词；
- temperature、top-p、最大输出、推理强度和停止序列；
- 可用工具范围；
- 默认权限模式；
- 默认启用的 Skills。

预设适合区分“纯聊天”“页面研究”“允许远端工具”等场景。删除预设不会删除已有会话记录，但新任务不能再选择它。Plugin 提供的预设是只读资产，需要先复制为用户预设才能编辑。

“后台任务模型”用于标题等低成本后台任务；留空时跟随默认模型。它不是独立对话模型，也不代表所有后台辅助功能都已接入。

## 常见错误

- **API Key 无效或权限不足**：检查 Key 是否能访问所选模型与接口。
- **模型不存在**：选择端点实际提供的模型 ID，必要时手工填写列表。
- **网络错误**：检查网络、代理、Base URL 和浏览器对端点域名的站点权限。
- **协议不兼容**：核对接口类型和兼容性选项；不要仅依据厂商宣传中的“兼容”字样。
- **上下文过长**：缩短引用内容、减少历史或新建会话。Panelot 不会为此自动删除你的历史记录。
